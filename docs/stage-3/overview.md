# Stage 3: Voice Evolution — Overview

Stage 3 is where voice is decided. A Stage 1 concept gives us a premise; a Stage 2 structure gives us a typed-edge plot DAG; what neither gives us is the *how* — the rhythm, register, narrator stance, syntactic texture, and prosody that make prose feel like itself rather than like every other competently-generated paragraph. Stage 3 produces that — locked, before Stage 4 writes a word.

The output of Stage 3 is a **voice specification per (concept, structure) pair**: a structured genome containing POV/tense, voice-mode shifts (for stories with shifting voice), free-form description of register and rhythm, positively-framed behavioral constraints, and — most importantly — three rendered prose passages on out-of-DAG scenes that demonstrate the voice in action. The renderings are not exemplars; they are the genome's load-bearing field.

This document covers the philosophy, the genome schema, the multi-agent session architecture, the adjacent-scene test bench, judging and selection, what carries over from earlier stages, and the handoffs on both ends. Operator prompts, rubric anchors, judge persona adaptation, and implementation details each live in their own docs.

## Philosophy: Voice Is the Texture of Attention

### Why a separate stage

Le Guin's *Steering the Craft* argues that rhythm, syntax, and sound are the *foundation* of prose quality, not decoration applied after the fact. The O'Sullivan stylometry study (Nature 2025) showed LLM outputs cluster tightly by model regardless of prompt — without explicit voice differentiation, every story generated by the same model sounds the same regardless of concept or structure. The Sui uncertainty-gap research (ICML 2026) shows human writing is 2–4× more informationally surprising than LLM output; an idiosyncratic voice spec pushes prose away from the model's generic default toward something more distinctive.

Without a separate voice stage, voice gets decided implicitly during prose generation. Implicit decisions drift toward defaults. The default for any current frontier LLM is the high register, past tense, single-POV, earnest stance documented in Lechmazur's flash-fiction benchmark across 29 models and 15,000+ stories. Stage 3 makes voice an explicit object that can be evaluated, iterated, and selected against — the same rigor we apply to concept (Stage 1) and structure (Stage 2).

### Why voice can't ride along with structure

The Stage 2 doc establishes that a faithful execution of a Stage 2 DAG can be written in many voices: Hemingway's "Hills" DAG can be rendered in Faulknerian maximalism and satisfy every node, edge, and motif. The structure underdetermines the voice. This is intentional — pushing voice into a separate search keeps each genome's evaluable surface narrow enough to judge rigorously. The cost is that Stage 2 alone cannot guarantee the prose will sound right; Stage 3 closes that gap.

### What Stage 3 is NOT for

- **Prose generation.** The renderings in the voice genome are 3 short passages on adjacent (out-of-DAG) scenes. Stage 4 writes the actual story prose.
- **Concept revision.** The concept is locked when it enters Stage 3.
- **Structural revision.** The DAG is locked when it enters Stage 3. Stage 3 may surface that a concept-structure pair cannot support a voice that satisfies the panel — that signal feeds the compost heap, not back into Stage 2 in v1.
- **Evaluating the voice as an abstract spec.** Voice is judged through its rendered prose, not through verbal description alone. Verbal description without rendered prose is *a verbal description of a voice with default-shaped prose*; it does not constitute a voice.

### The voice spec is the load-bearing handoff to Stage 4

Stage 4 will inject the voice genome into prose generation. The voice genome's `description`, `rhythm`, `diction`, `positive_constraints`, and `prohibitions` fields condition the generator; the `renderings` field anchors the cache prefix as few-shot exemplars. If Stage 3 produces a generic voice, Stage 4 produces generic prose, and the entire pipeline collapses to model defaults regardless of how strong the concept and structure are. This is why Stage 3 gets a dedicated multi-agent generation session per (concept, structure) pair, not a cheap classifier or mutation operator.

## The Genome: A Voice Specification

### Schema

The schema is a **thin discrete shell with free-form prose primary**, plus five structured fields drawn from literary theory that LLMs systematically default badly on. Each enum field is small token cost; collectively they push the model off its near-default attractor on dimensions that the verbal description alone won't reach.

```yaml
voice_genome:
  pair_id: "c_a8f12e_struct_0"      # reference to (concept, structure) pair
  agent_id: "the-reductionist"      # which voice agent produced this proposal
                                    # (winner field; archive keeps all 4)

  # ─── Discrete shell: narrating instance ───────────────────────────
  pov: "third"                      # required: first | second | third
  pov_notes: ""                     # free-form — close vs. omniscient drift,
                                    # second-person interior, hybrid handling
  tense: "past"                     # required: past | present
  tense_notes: ""                   # free-form

  # ─── Multi-mode support (for shifting / dual voice) ───────────────
  voice_modes: []                   # list of mode definitions for stories
                                    # whose voice changes; see below
  voice_dynamics: ""                # free-form: voice trajectory across the
                                    # DAG; e.g. "begins breezy and distancing,
                                    # collapses to clipped immediacy by climax"

  # ─── Lit-theory fields (Genette / Cohn / Bakhtin / Booth / Le Guin) ──
  # These five are the highest-leverage structured fields per the
  # voice-literary-theory research. LLMs systematically default badly on
  # each. Full 30-field v2 schema deferred — see lab/deep-research/runs/
  # 2026-04-27-voice-literary-theory/report.md for what's in v2.

  consciousness_rendering:          # Cohn 1978: how the prose handles
                                    # the character-narrator boundary.
                                    # Default LLM behavior is psycho-narration;
                                    # this field forces an explicit choice.
    mode: "narrated_monologue"      # psycho_narration | narrated_monologue |
                                    # quoted_monologue | mixed
    fid_depth: "deep"               # none | shallow | deep | dominant
                                    # Free indirect discourse depth — the
                                    # single highest-leverage field.

  implied_author:                   # Booth 1961: the second-self constructed
                                    # from aesthetic choices. Distinct from
                                    # narrator. Sets moral temperature.
    stance_toward_characters: "elegiac"  # compassionate | ironic | contemptuous
                                         # | elegiac | satirical | neutral
    moral_temperature: "warm"       # warm | cool | cold | ambiguous

  dialogic_mode:                    # Bakhtin: whether the narrator's language
                                    # is infected by social registers.
                                    # Orthogonal to focalization.
    type: "heteroglossic"           # monologic | heteroglossic | polyphonic

  craft:                            # Le Guin 1998 ch. 3, 7: practitioner
                                    # dimensions absent from academic narratology.
    sentence_rhythm: "varied"       # short_dominant | varied | long_dominant
                                    # | staccato | sinuous
    crowding_leaping: "leaping"     # crowded | balanced | leaping
                                    # Density per prose unit. LLMs default
                                    # to balanced; explicit choice matters.

  # ─── Free-form prose description ──────────────────────────────────
  description: |                    # the voice's relationship to its subject,
                                    # not "what kind of person" — narrator
                                    # stance, register, attitude toward
                                    # material. Should align with implied_author
                                    # fields above; this is the prose form.
  diction: ""                       # free-form: vocabulary register, level
                                    # of abstraction, characteristic moves.
                                    # The structured `craft.sentence_rhythm`
                                    # subsumes the previous free-form rhythm
                                    # field; rhythm-related details that don't
                                    # fit the enum belong here as notes.

  # ─── Constraints (asymmetric: positive primary, negative secondary) ──
  positive_constraints:             # POSITIVELY-FRAMED behavioral specs
    - "..."                         # e.g. "grounded in concrete physical
                                    # action; figurative language reserved
                                    # for moments of structural rupture"
  prohibitions:                     # NEGATIVELY-FRAMED, low-pressure or
                                    # trivially-checkable only
    - "..."                         # e.g. "the word 'felt' never appears"

  # ─── Renderings (load-bearing) ────────────────────────────────────
  renderings:                       # 3 voice-transformed adjacent scenes;
                                    # the proof of the voice
    - scene_id: "morning_kitchen"
      text: "..."
    - scene_id: "argument_at_door"
      text: "..."
    - scene_id: "after_the_news"
      text: "..."
```

### Why these five lit-theory fields specifically

The voice-literary-theory research (`lab/deep-research/runs/2026-04-27-voice-literary-theory/`) surveys the full apparatus — Genette's narratology (focalization, narrating time, pace, frequency), Bakhtin's heteroglossia, Cohn's consciousness modes, Booth's implied author, Halliday's tenor, Le Guin's craft framework. The full v2 schema in that report has ~30 fields across 8 categories. We adopt only 5 in v1 because:

1. **`consciousness_rendering.fid_depth`** — Free indirect discourse is the deepest single discriminator between literary styles. LLMs default to psycho-narration ("She felt the weight of what he hadn't said") and require explicit specification to produce narrated monologue ("How could she possibly be ready in time?"). FID is what Wood calls "the novelist's instrument of compassion." Auto-evaluation is unreliable (F1 ~0.45-0.5 per Brunner et al. 2019); pairwise judge eval is the right verification.

2. **`implied_author.stance_toward_characters`** — Booth's central concept: the implied author is constructed from every aesthetic choice and is distinct from the narrator. Without an explicit stance field, LLMs produce affectively neutral narrators. The Saunders/Gardner/Wood convergence ("voice is the moral temperature of a story") makes this field load-bearing for readability and resonance.

3. **`dialogic_mode.type`** — Bakhtin's heteroglossic axis is orthogonal to focalization. A narrator's language can be socially infected (multiple registers in tension) regardless of how many characters are focalized. Marking it explicitly as monologic vs. heteroglossic vs. polyphonic forces the agent to think about *how* the narrator's voice carries social registers, not just *whose perspective* the narrator uses.

4. **`craft.sentence_rhythm` (refined to enum)** — Le Guin's ch. 3 vocabulary. Most measurable of the craft dimensions; auto-evaluable via burstiness in Tier-A pre-screening. The previous free-form `rhythm` field is replaced by this enum + `diction` notes for residue.

5. **`craft.crowding_leaping`** — Le Guin's most original contribution (ch. 7). LLMs default to balanced/predictable density. This is the dimension on which voice transformation produces the most measurable departure from neutral baseline. Crowding accumulates sensory/cognitive detail; leaping uses ellipsis and reader inference; balanced is the default attractor.

The fields *not* adopted in v1 (Genette's complete focalization typology, narrative levels, narrating time, pace, frequency; Halliday's tenor decomposition; epistemic authority type; etc.) are deferred until pilot data shows whether the v1 schema produces sufficient differentiation. Adding fields when they aren't needed costs attention budget — Principled Personas EMNLP 2025 documented up to 30pp performance drops from irrelevant attributes — and the project's coding philosophy explicitly favors "no premature abstraction." Revisit in v1.5 with empirical signal.

### Multi-mode voices

For stories with explicit voice shifts (Munro's tense alternation, Calvino's nested registers, Cloud Atlas's six prose registers, Saunders' character-voice-absorption), the `voice_modes` field lists alternative configurations:

```yaml
voice_modes:
  - name: "memory_register"
    condition: "passages of remembered past time"
    overrides:
      pov: "first"
      tense: "present"
      diction_notes: "lyric, fragmentary, image-led"
  - name: "present_action"
    condition: "current-time exterior action"
    overrides:
      pov: "third"
      tense: "past"
      diction_notes: "spare, declarative, dialogue-heavy"
```

The `voice_dynamics` free-text field captures voice trajectories that aren't cleanly mode-switchable — voice that gradually compresses from breezy to clipped as dread accumulates, or that absorbs character voice over the course of the story. Bakhtin's heteroglossia is the literary-theory grounding here; most novels are dialogic, not monologic, and the schema needs to handle that.

### Why renderings are the genome

Verbal description alone is unreliable. The voice-as-evolvable-artifact research (`lab/deep-research/runs/2026-04-27-voice-as-evolvable-artifact/`) is convergent: LLMs capture surface stylistic tics from verbal descriptions but miss the texture that constitutes genuine voice. GPT-4o-Hemingway has 38% *more* lexical diversity than actual Hemingway; the model imitates its representation of the cultural cliché, not the actual prose distribution.

Few-shot demonstrated examples outperform verbal descriptions in style transfer (Wang EMNLP 2025; Mikros 2025). Authorship-embedding-conditioned generation outperforms GPT-4 prompting on style fidelity (TinyStyler EMNLP 2024). The implication for Stage 3: the voice genome's verbal fields are documentation of the voice; the renderings are the voice itself, and selection must operate on the renderings.

A mutation that produces a new verbal description without producing new renderings is not a mutation in voice space — it's a mutation in the *labeling* of a voice. Renderings are required at every commit point.

### Constraint asymmetry

`positive_constraints` is the primary field. `prohibitions` is the secondary field, reserved for low-pressure or trivially-regex-checkable items.

The mechanistic basis: Rana 2026 quantified violation rates for negative constraints. With the formula p = σ(−2.40 + 2.27·P₀), where P₀ is the baseline probability of the prohibited element:
- P₀ = 0.1 (rarely-generated): ~9% violation
- P₀ = 0.5: ~24% violation
- P₀ = 0.9 (default behavior): ~46% violation

87.5% of violations are *priming failures* — naming the forbidden element activates rather than suppresses it in the model's attention. "No metaphors" in a literary fiction context is exactly the high-pressure case where this fails. The fix: positive reframing. "Grounded in concrete physical action; figurative language reserved for moments of structural rupture" produces lower violation rates than "no metaphors."

Reserve literal "never use X" prohibitions for elements with low baseline probability (e.g., the word "abortion" in a Hemingway-mode constraint story — a low-frequency word the model isn't pushing toward) or trivially regex-checkable items where post-generation validation is cheap.

## The Multi-Agent Session

Stage 3 runs a structured multi-agent generation session per (concept, structure) pair. Four voice agents — each with a distinct ordinary-specific persona — develop voice proposals in parallel under controlled visibility, then converge through a single-pass critique and revision phase, then vote.

### Why multi-agent, not evolutionary

Voice spec is a small near-discrete artifact. Stage 1's population-based ShinkaEvolve and Stage 2's tree-search MCTS would be overkill. Multi-agent generation maps cleanly to the cognitive-science finding that nominal-then-share beats both purely-individual and free-collaborative ideation [Diehl & Stroebe 1987 on production blocking; Linsey & Becker 2010 on brainwriting; Zhou et al. 2019 on incubation intervals]. The published Spark Effect architecture (Doudkin et al. 2025) — which is structurally identical to what we propose — measured +4.1 points on a 1-10 diversity scale over uniform prompting in creative ideation tasks.

The "binding infrastructure is judge panel + pairwise protocol + compost heap, not a shared search algorithm" principle from `docs/stage-2/overview.md` §"How Stage 2 Differs From Stage 1" continues to apply. Each stage gets the right engine for its genome.

### The four voice agents

Personas live in `configs/voice_agents/`. Each is an *ordinary-specific aesthetic-commitment persona* — not a famous-author archetype. The Deng/Brucks/Toubia 2026 finding (`lab/references/persona-knowledge-partitioning/`): ordinary specific personas produce 26% more unique combinations than celebrity personas, because they push the model into under-sampled corners of the prior. Famous-author personas (CoMPosT EMNLP 2023; Mikros 2025) collapse to caricature.

The four are designed to be orthogonal on the four axes most relevant to voice differentiation:

| Persona | Interiority | Time | Reader | Surface |
|---|---|---|---|---|
| **the-reductionist** | suppress | compress | withhold | plain |
| **the-temporal-collagist** | center via memory | nonlinear | withhold via gaps | variable |
| **the-sensory-materialist** | redirect to objects | real-time | make physical | plain-but-textured |
| **the-structural-experimentalist** | form is content | form-determined | withhold via opacity | variable |

Each persona has:
- A specific non-writer background (insurance adjuster / archivist / forensic entomologist / book designer-architect) that grounds the aesthetic stance in material work
- 3–5 ranked aesthetic commitments with rationale
- 1–3 explicit aversions
- 2–3 productive obsessions
- An explicit epistemic-skepticism mandate (countering sycophancy, which correlates with persona agreeableness at r=0.87 per Shah et al. 2026)
- 1–2 first-person demonstrations of the voice's craft moves in working mode

See `configs/voice_agents/_template.yaml` for the schema and the design rationale.

### The five-phase flow

Inter-agent visibility is **enforced architecturally**, not via soft instruction. Same-model panels show cosine similarity 0.888 in reasoning chains even with role-prompt variation [Patel et al. 2026 on representational collapse]. Structural independence during generation is a hard requirement, not a preference.

#### Phase 1 — Private brief

Each agent receives:
- The Stage 1 concept genome
- The Stage 2 structure DAG (rendered)
- The 3 adjacent-scene neutral drafts (see §"Adjacent Scenes" below)
- Their own persona system prompt
- Tool access (see §"Tool Surface" below)

They generate independently. CoT reasoning permitted and encouraged. Output: a complete voice spec proposal including all 3 renderings.

**Visibility:** Each agent cannot see other agents' work, personas, or progress.

#### Phase 2 — Private judge consultation

Each agent is allowed up to 3 questions per judge persona. Questions are constrained to taste/principle/aesthetic-method ("how do you approach figurative language in moments of grief?", "what tells you a metaphor has earned its place?"). NOT score predictions on the candidate, NOT rewrite requests. Tool-level enforcement (regex + system prompt; see §"Tool Surface").

Output: agents may revise their Phase 1 proposal in light of judge taste responses. The revision is private.

**Visibility:** Each agent's queries are private. Agents cannot see who consulted whom or with what.

#### Phase 3 — Reveal + structured single-pass critique

All four final Phase-2 proposals are revealed simultaneously. Each agent reads the other three proposals and writes a fixed-shape critique on each:

```yaml
critique:
  target_agent: "the-temporal-collagist"
  strengths:
    - "..."
    - "..."
  concern: "..."
```

No dialogue. No follow-ups. No multi-turn debate. The conformity literature is convergent — multi-turn debate causes opinion shift and authority-gradient deference [Choi et al. ACL 2025 on 2,500+ debates; Nature Sci Reports 2026 on adversarial persuasion]. Single-pass structured critique extracts the cross-pollination benefit while limiting conformity risk. Wu & Ito EMNLP 2025 call this "implicit consensus" — agents share information then decide independently — and find it outperforms explicit-consensus structures in dynamic environments.

**Visibility:** All 4 final proposals visible. Critiques are written privately and aggregated to the next phase, not exchanged in real time.

#### Phase 4 — Independent revision

Each agent receives the critiques targeting its own proposal. Performs ONE private revision pass. May fully adopt, partially adopt, or reject any critique. The revised proposal is the agent's final commit.

**Visibility:** Own proposal + critiques received. Cannot see others' revisions.

#### Phase 5 — Borda vote (no self-vote)

Each agent ranks the other 3 final proposals from best to worst on overall voice quality. Borda count aggregates: rank-1 = 2 points, rank-2 = 1 point, rank-3 = 0 points.

The result is the **writers' verdict** — recorded as metadata. It does not drive selection. See §"Selection" for how it interacts with the readers' verdict.

Borda is preferred over Condorcet/Schulze for small panels; the LLM-Council practitioner analysis and Yang et al. AIES 2024 both establish that with 3-5 LLM voters, Schulze methods amplify noise and produce cycles, while Borda provides a stable signal from average rank.

### Diversity monitor

Pairwise embedding distance of agent proposals is measured at end of Phase 1, end of Phase 2, end of Phase 3 (post-reveal, pre-revision), and end of Phase 4 (final). If the pairwise embedding distance falls below threshold T (TBD via pilot — initial guess: cosine distance < 0.3 across all pairs), the session is restarted with more heterogeneous personas or different model-family assignments. This catches the representational-collapse failure mode early instead of paying for the full session and producing a degenerate vote.

The metric is also archived for cross-pair analysis: if pairs systematically collapse below threshold, the persona set or model-family assignment needs revision globally.

## Adjacent Scenes — The Fixed Test Bench

Voice candidates must compete on equal terrain. If each candidate generates renderings of *its own* preferred scenes, the comparison becomes "which voice picked the easier showcase," not "which voice is stronger." Resolution: a fixed test bench of three scenes, generated once per (concept, structure) pair and shared across all four voice agents.

### Generation flow

1. **AU point extraction.** A smart-LLM-with-reasoning call receives the Stage 1 concept and Stage 2 structure DAG and picks three "fitting alternative-universe points" — moments in the same fictional world, with the same characters, that are NOT beats in the DAG. The picker chooses three scenes that **stress-test distinct demands on voice for *this* story**, articulating each demand in its own taxonomy and justifying why the three differ from one another. The articulated demands travel with each scene record.

   We deliberately do not prescribe a fixed register triad (e.g. low-tension / dialogue / climactic). The load-bearing principle is "stress-test voice across distinct demands"; what counts as a distinct demand is concept-dependent. A quiet domestic story may have no meaningful climactic register; a Beckett-mode interior may have no useful dialogue scene. Common demand axes the picker may draw on — but is not required to use — include tonal range, dialogue vs. interior, public vs. private, action vs. contemplation, intimate vs. distanced, sensory density, speed of cognition, pace of disclosure. The articulation step is a guard: if the three "demands" come back as paraphrases, that's a flag for inspection rather than a silent failure.

2. **Neutral-voice drafting.** A follow-up call generates flat, content-conveying default-LLM-register drafts of each picked scene. ~150–300 words each. Explicit instruction to write in neutral default register; no stylistic flourishes; no figurative language; clean expository prose. These drafts are the baseline against which voice transformations are measured.

3. **Caching.** Drafts and articulated demands are cached at session start and shared as fixed input across all 4 voice agents.

### What agents do with the drafts

Each voice agent's task in Phase 1 is to **transform** the three neutral drafts in their proposed voice. They do not regenerate content; they re-render style. Same scene, same events, same characters — different voice.

This isolates voice from content. Stylometric distance from the neutral baseline is then a direct measurement of voice distinctiveness. Pairwise judge comparison operates on the rendered voice transformations against each other, with the neutral baseline as a third reference point.

### Why three, not one

A voice that handles one demand beautifully but collapses on a different demand is a partial voice. Three scenes covering distinct demands stress-test the voice across the load conditions that matter for this story and surface the cases where a voice's distinctiveness is brittle.

If pilot data shows three is insufficient (voices fragmenting at boundaries between the chosen demands) or excessive (signal redundant across the three), v1.5 may revise the count. The "demands are paraphrases" failure mode — where the picker articulates three demands that read as one demand in different words — is the leading risk; if it shows up in pilots, the picker prompt gets stronger cross-demand-justification scaffolding.

### Why "adjacent" specifically

Why not generate test passages from beats in the actual DAG? Two reasons:

1. **Stage 4 still needs to write the prose.** Stage 3's renderings cannot pre-spoil the actual story or pre-commit Stage 4 to specific phrasings. AU framing (same world, different scenes) gives voice agents freedom to demonstrate without contaminating the downstream task.

2. **The DAG beats are constrained by structural payload requirements.** A beat that has to enact a specific causal payload, motif mode, and disclosure constraint is over-determined; the voice has less latitude. AU scenes free the voice from structural payload obligations, isolating voice from structure.

## Tool Surface

Each voice agent operates with a constrained tool set during the session. Tools are intentionally bounded — agents are not autonomous research assistants; they are voice generators with controlled inputs.

### `ask_judge(judge_id, question)`

Query a judge persona for taste/principle/method input.

**Allowed:** Questions about the judge's taste, evaluative principles, what they look for, what they generally flag, examples from their canon. Generative questions ("what makes a metaphor earn its place in your view?") rather than evaluative ones ("would you score this 4/5?").

**Blocked:**
- Score predictions or evaluative claims about a specific candidate (regex on `score|rate|grade|would you give|how would you mark` etc., plus system-prompt instruction in the judge persona to refuse).
- Rewrite requests ("write me a better version of this passage"). Judges are calibrated for evaluation, not generation.

**Why constrained:** Allowing score-prediction queries causes Goodhart-style optimization — agents learn to score-hack rather than write well, and the judge becomes a target rather than a measure. Allowing rewrite requests substitutes the judges' (worse) prose for the writer-agents' (better) prose.

See `feedback_judge_query_scope.md` memory.

### `lookup_reference(query)`

Search a curated literary-reference corpus.

**Corpus:** `data/voice-references.yaml` (TBD), public-domain heavily, brief modern excerpts (≤500 words) under research-use fair-use. Tagged by register / era / mode / specific voice technique (free-indirect-discourse, second-person, present-tense interior, etc.). See `project_research_not_product.md` memory for legal posture.

**Returns:** Full passage + metadata (author, title, era, register tag, technique tag).

### `thesaurus(word, mode)`

Datamuse API queries. Free, no API key.

| Mode | Datamuse code | Use case |
|---|---|---|
| `means_like` | `ml` | synonyms / semantic neighbors |
| `sounds_like` | `sl` | phonetic similarity (sound work) |
| `related_to` | `rel_trg` | words that often co-occur |
| `adjective_for` | `rel_jjb` | adjectives commonly modifying a noun |
| `noun_for` | `rel_jja` | nouns commonly modified by an adjective |
| `antonyms` | `rel_ant` | opposites |
| `topics` | `topics` | filter by subject area |

Cache aggressively; queries are deterministic. Useful particularly for diction work and for the sensory-materialist persona's specificity-pressure operations.

### `render_adjacent_scene(scene_id)`

Retrieve the cached neutral-voice draft for one of the 3 adjacent scenes. Read-only; the rendering pool is fixed at session start.

### `stylometry(passage, caller_model=None, neutral_baseline=None)`

Compute content-independent stylometric signals on a candidate prose passage and return them alongside aggregate references for positioning. Implementation: `owtn/tools/stylometry.py + owtn/tools/lookup_exemplar.py`.

**Signals on the candidate** (raw, interpretable units):
- `burstiness` — coefficient of variation of sentence length. Heuristic: <0.4 reads as LLM-flat; >0.4 is human-like; >0.6 is varied.
- `mattr` — moving-average type-token ratio (lexical diversity). Caveat: unreliable alone (Mikros 2025 — GPT-4o-Hemingway has higher MATTR than actual Hemingway).
- `fw_distance_from_own_model_default` — function-word cosine distance from the calling model's default centroid. Primary "departure from your own attractor" signal.
- `fw_distance_from_llm_centroid` — distance from the cross-model LLM centroid.
- `fw_distance_from_human_literary` — distance from the literary canonical centroid.
- `fw_distance_from_neutral_baseline` — distance from the session's neutral-voice draft, when provided. Stage 3 orchestrator passes this in active sessions; standalone tool calls receive None.

**References** (aggregate centroids; ~5 rows, total response < 4kB):
- `caller_model_default` — mean signals over the caller's model samples (Sonnet 4.6, DeepSeek V4-pro, etc.). Falls back to a "model not in corpus" stub when the caller model is unknown.
- `llm_default_centroid` — mean across all LLM-default entries (currently 5 models × 5 prompts from EQ-Bench, plus DeepSeek V4-pro on the matching prompts).
- `human_literary_centroid`, `human_amateur_centroid`, `expository_centroid` — register anchors from the curated literary, WritingPrompts amateur, and Wikipedia expository entries.

**Interpretation notes** — short dynamic guidance derived from the candidate's actual values (e.g., "Burstiness 0.32 is below the 0.4 floor — sentence rhythm reads as LLM-flat. Vary sentence length.").

**Use posture (advisory, not gating).** This tool returns signals an agent reads when revising; the orchestrator does not auto-reject candidates based on these numbers. Threshold calibration is unsolved in v1; agent-as-consumer lets the signal be useful immediately as guidance. See `lab/issues/2026-04-27-stylometric-tool-for-voice-agents.md`.

**Reference corpus.** `data/voice-references-stylometric.yaml` (metadata) + `data/voice-references/passages/<id>.txt` (prose). Bootstrapped from the user's TRIBE v2 prose-pilot dataset and EQ-Bench's creative-writing-bench v3 run dump. Pre-computed signals live in `data/voice-references-stylometric.cache.json`, invalidated by SHA-256 on each passage. Add new entries by appending to the YAML and dropping the prose into `passages/<id>.txt`.

**LIWC and authorship-embedding signals** are stubs in v1 (per `lab/issues/2026-04-27-stylometric-tool-for-voice-agents.md`); wire when Stage 4/5 prose evaluation needs them.

### `note_to_self(text)`

Scratchpad. Private to the agent. Persists across phase transitions. Not visible to any other agent or to the orchestrator's logs (except for debugging).

### Tools agents do NOT have

- Direct access to other agents' work, scratchpads, or judge queries (until Phase 3 reveal)
- Web search, code execution, or any open-ended network access
- The ability to modify the adjacent-scene neutral drafts (these are fixed)
- The ability to call other voice agents directly

## Selection — Two Verdicts

### Readers' verdict (decides)

The 4 contest judges (`gwern`, `roon`, `alexander-wales`, `jamie-wahls`) run pairwise comparison on the rendered voice transformations of the 3 adjacent scenes. Same per-criterion protocol as Stage 1/2. Voice-specific rubric anchors live in `docs/stage-3/rubric-anchors.md` (TBD); core dimensions:

- **Distinctiveness** — does this voice sound different from generic LLM prose? Is the authorial fingerprint legible?
- **Internal consistency** — does the spec hold together across the three rendered scenes? Or does the voice fragment under load?
- **Appropriateness** — does this voice serve the concept and the structure's emotional shape?
- **Anti-slop adherence** — does the rendered prose avoid the documented LLM voice patterns (Tier-A signals)?
- **Constraint enactment** — are the spec's positive constraints actually enacted in the renderings, or merely declared?

The readers' verdict drives selection — matches the project's "did this story grip you" north star.

### Writers' verdict (metadata)

The Borda count from Phase 5 of the multi-agent session. Recorded as metadata; on close pairwise calls (within ε), it can break ties.

Divergence between writers' and readers' verdicts is itself signal. Persistent disagreement might indicate either:
- The judges are calibrated for resonance / reader experience while the writers track craft moves the judges underweight (treat as informative diversity)
- The writers have learned each other's tells and are voting on something orthogonal to actual voice quality (calibration concern; investigate)

### Stylometric pre-screening

Before the expensive judge tournament, candidates pass through automated screening (Tier-A-like, but voice-specific):

- **Burstiness** (CV of sentence length) — proxy for the full craft cluster (voice/imagery/rhythm correlation ~0.96–0.99 per Lechmazur 2025).
- **Function-word z-scores** — Burrows' Delta foundation; content-independent register/stance signal.
- **LIWC psychological dimensions** — best matched-content authorship classification accuracy in Mikros 2025 (84%).
- **Authorship-embedding distance from model-default centroid** — direct measurement of departure from default.

Selection target is **departure from model default**, not proximity to any named author. Critical principle: select on "stylometrically distinct from neutral baseline" not "sounds like Hemingway"; the latter converges on cultural pastiche [Mikros 2025: GPT-4o-Hemingway has 38% MORE lexical diversity than actual Hemingway — emulation distorts].

A candidate scoring near-default on all four signals is filtered before judging — it's a verbal description of a voice with default-shaped prose, and the expensive judge panel won't recover what stylometric pre-screening already showed isn't there.

Thresholds calibrated empirically from baseline neutral-voice drafts vs. voice-transformed drafts (TBD pilot).

## How Stage 3 Differs from Earlier Stages

Stage 3 runs on a different engine than Stages 1 and 2, produces a different genome, and uses different selection mechanics. What carries over is the judge panel, the pairwise-comparison philosophy, the LLM-client infrastructure, and the compost heap.

| Aspect | Stage 1 | Stage 2 | Stage 3 |
|---|---|---|---|
| Engine | ShinkaEvolve (async population, islands) | MCTS (per-concept tree per preset) | Phased multi-agent session per (concept, structure) |
| Genome | Concept JSON | Typed-edge DAG JSON | Voice spec JSON with renderings |
| Population unit | Concept across islands | DAG nodes/edges within trees | Agent proposals within session |
| Diversity engine | Islands + operator bandit | Pacing presets | 4 ordinary-specific personas (knowledge partitioning) |
| Selection | Pairwise champion succession + Swiss tournament | UCB1 + within-concept tournament | Pairwise judge tournament + agent Borda metadata |
| Compost feed | Yes | Yes (shared with Stage 1) | Yes (loser proposals + critiques to shared compost heap) |
| Cost (light) | $5–$10 / run | $30–$48 / run on top of Stage 1 | $5–$20 / run on top of Stage 2 (provisional; pilot to validate) |

### What carries over

- **LLM client** (`owtn/llm/`) — caching, multi-provider support, cost tracking, bandit ensemble
- **Judge panel** (`configs/judges/`) — same 4 contest judges, with voice-specific rubric anchors. Per-judge translation concerns covered in `docs/stage-3/judges.md` (TBD).
- **Pairwise protocol** (`owtn/evaluation/pairwise.py`) — same per-criterion voting mechanism applied to voice rendering pairs
- **Compost heap** — losing voice proposals, archived critiques, and rejected renderings feed the same SQLite compost table as Stages 1 and 2

### What does not carry over

- **ShinkaEvolve engine.** No population evolution; voice spec is not the right genome for it.
- **MCTS.** No tree search; no obvious tree topology for voice space.
- **Islands or pacing presets.** Diversity comes from agent personas + adjacent-scene fixed bench + stylometric pre-screening.
- **MAP-Elites archive.** Voice space doesn't have obvious orthogonal axes for grid archiving in v1. May be revisited in v1.5 if pilot data suggests useful axes (departure-from-default, distinctiveness, register, voice-modes-count).

## Handoff From Stage 2

Stage 3 receives for each (concept, structure) pair:

1. **Complete structure DAG** — all nodes, edges, payload fields, motif threads, character arcs, story constraints, concept demands.
2. **Tournament ranking** — within-concept tournament result (1st/2nd/… among presets).
3. **QD archive metadata** — which (disclosure_ratio, structural_density) cell the structure occupies.
4. **Pacing preset label** — which preset generated this structure.
5. **Forwarded Stage 1 concept** — required for voice appropriateness judgment (style_hint, target_effect, affective_register, literary_mode).
6. **Adaptation permissions** — present in handoff manifest but named-only in v1 (per Stage 2's pattern). No re-entry mechanics in Stage 3 v1.

### How Stage 3 uses the handoff

The (concept, structure) tuple is injected into the multi-agent session at Phase 1 setup. Each agent receives:
- The full concept (premise, target_effect, character_seeds if any, thematic_tension, constraints, style_hint)
- The structure DAG rendered in the same incident-encoded format as Stage 2's expansion prompts
- The 3 adjacent-scene neutral drafts (generated once at session start from the (concept, structure) pair)

Concept's `target_effect` and `thematic_engine` are wrapped with the goal-field discipline from `docs/prompting-guide.md` §"Goals Cannot Ride in Plaintext" — these are narrative goal fields that must not be passed verbatim into voice generation prompts (the LLM will literalize them into the renderings). Wrap with explicit instruction not to enact them as plain text.

### Top-K policy

`light.yaml`: K=1 structure per concept (minimum cost; one Stage 3 session per concept).
`medium.yaml`: K=2 (preserves the largest within-concept structure-diversity gap).
`heavy.yaml`: K=all (maximum diversity to Stage 4).

Each surviving structure produces ONE voice winner from its session. Stage 4 receives N concepts × K structures × 1 voice winner = N×K complete (concept, structure, voice) tuples.

---

## Handoff to Stage 4

Stage 4 (prose evolution) receives for each advancing (concept, structure, voice) tuple:

1. **Complete voice genome** — all fields, including renderings, post-revision.
2. **Adjacent scene drafts** — both the neutral baseline and the winner's rendered transformations. Stage 4 may use these as in-context exemplars.
3. **Stylometric pre-screening signals** — for monitoring whether Stage 4's prose maintains voice fidelity (drift detection).
4. **Forwarded Stage 1 + Stage 2 artifacts** — concept, structure, all upstream metadata.
5. **Adaptation permissions** — named only in v1.

### How Stage 4 uses the voice genome

The voice spec is injected as a **system-prompt block** in Stage 4's prose generation. The genome's `description`, `rhythm`, `diction`, `positive_constraints`, and `voice_modes` fields condition the generator. The renderings populate the prompt-cache prefix as few-shot exemplars (significantly cheaper than re-injecting per scene).

Constraint precedence on conflicts:
- Stage 1 `constraints` (concept-level prohibitions) take priority over Stage 3 `prohibitions` (voice-level prohibitions). If they conflict, the candidate voice fails Tier-A validation before reaching judges.
- Stage 1 `style_hint` is informational; Stage 3 voice spec is operational. Style hint informed Stage 3 generation; in Stage 4, the voice spec wins.
- Stage 2 `story_constraints` (diegetic prohibitions) and Stage 3 voice prohibitions live in different namespaces. Both apply.

## Validation Protocol

Voice genomes pass through three gates before reaching the judge tournament.

### Gate 1: Schema validation

- All required fields present (pov, tense, description, rhythm, diction, positive_constraints, renderings)
- `pov` ∈ {first, second, third}; `tense` ∈ {past, present}
- `renderings` has exactly 3 entries, each with non-empty text
- `voice_modes` (if present) all reference valid POV/tense values
- `pair_id` references a valid Stage 2 winner
- `agent_id` is one of the configured voice agents

### Gate 2: Constraint coherence

- Stage 1 concept `constraints` are not contradicted by Stage 3 `prohibitions` or `positive_constraints` (this is the cross-stage conflict check)
- Listed `prohibitions` do not name high-pressure prose elements (regex check against a configured list of high-P₀ words: metaphor-related vocabulary, common figurative-language signal terms, etc. — encourage positive reframing)
- Trivially-checkable prohibitions (e.g., "the word X never appears") are validated against the renderings: regex pass over each rendering must confirm compliance

### Gate 3: Stylometric pre-screening

Render each of the three voice transformations and compute:
- Burstiness (CV)
- Function-word z-scores (deviation from neutral baseline)
- LIWC psychological dimensions (deviation from neutral baseline)
- Authorship-embedding distance from model-default centroid

A candidate that scores near-default on all four signals fails Gate 3. Thresholds set empirically from pilot data.

Genomes failing any gate are rejected before judge tournament; their reasoning chains and the failure reason are archived.

## Open Questions

These are flagged for resolution during pilot or in subsequent docs:

1. **Adjacent-scene picker reliability.** Does the smart-LLM-with-reasoning call reliably pick three AU points placing distinct demands on voice, or do the three articulated demands collapse to paraphrases? Picker design and implementation in `lab/issues/2026-04-28-adjacent-scene-picker.md`. The articulated-demands field plus cross-demand justification step is the v1 guard; pilot will show whether stronger scaffolding is needed.

2. **Multi-mode voice generation in practice.** When `voice_modes` are present, do agents reliably produce renderings that demonstrate the mode shifts? Or do they collapse to the dominant mode?

3. **Stylometric thresholds.** What does "near-default" mean numerically for each of the four signals? Empirical distribution from pilot needed.

4. **Judge persona translation for voice.** The 4 contest judges are calibrated for concept and structure judging. Voice-specific judging may need adapted rubric anchors. Per-stage rubric-anchors doc (`docs/stage-3/rubric-anchors.md`) TBD.

5. ~~**Aestheticized session framing.**~~ **RESOLVED.** Workshop frame, not mystical theater. The mystical/theatrical attractor IS measurably real (Lu et al. 2026 "Assistant Axis") but ornamental framing causes Waluigi-mode caricature collapse. Concrete prescription:
   - Eliminate standard assistant-register vocabulary from agent system prompts ("I am here to help," "as a creative writing assistant")
   - Wrap the session in a minimal workshop frame: "we are here to make this work, not to perform correctness; our criterion is whether the prose is alive"
   - Do NOT add evocative register theater (no "scriptorium," no "council of voices," no "alchemists of language")
   - Each prompt repositions the model on the Assistant Axis (R²=0.53–0.77 per Lu et al.) — keep that in mind for prompt-drafting decisions
   See `lab/deep-research/runs/2026-04-27-aestheticized-prompt-framing/report.md`.

6. ~~**Literary-theory schema vocabulary refinement.**~~ **RESOLVED.** 5 high-leverage fields adopted in v1 (`consciousness_rendering.fid_depth`, `implied_author.stance_toward_characters`, `craft.crowding_leaping`, `craft.sentence_rhythm`, `dialogic_mode.type`). Full 30-field v2 schema deferred until pilot data justifies it. See schema section above and `lab/deep-research/runs/2026-04-27-voice-literary-theory/report.md`.

7. ~~**Agent framework decision.**~~ **RESOLVED.** Roll-our-own thin orchestration over `owtn/llm/`. No framework evaluated enforces inter-agent visibility boundaries architecturally. Module structure in `owtn/voice/`: orchestrator.py, phases.py, state.py, tools.py, borda.py, tracing.py. Estimated 330–460 LoC. See `lab/deep-research/runs/2026-04-27-agent-framework-comparison/report.md`.

8. **Cost validation.** Provisional estimate $5–$20 per session at light config. Pilot needed to confirm.
